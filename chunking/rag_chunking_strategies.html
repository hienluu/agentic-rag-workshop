RAG Chunking Strategies

Understanding different approaches to document segmentation in Retrieval-Augmented Generation

C
Character-based Chunking
Splits text based on fixed character count limits, creating chunks of uniform size. This is the simplest approach, focusing purely on length constraints.
The quick brown fox jumps over the lazy dog. This
is an example of character-based chunking where
each chunk has exactly 50 characters including sp
aces and can break mid-word or mid-sentence.

ADVANTAGES

  • Predictable chunk sizes
  • Simple implementation
  • Consistent memory usage
  • Language agnostic

DISADVANTAGES

  • Breaks semantic meaning
  • Can split words/sentences
  • Poor retrieval quality
  • Context loss at boundaries
R
Recursive Chunking
Uses a hierarchical approach, trying different separators (paragraphs → sentences → words → characters) until chunks fit within size limits while preserving semantic boundaries.
The quick brown fox jumps over the lazy dog.
This is an example of recursive chunking.
It respects sentence boundaries first, then paragraphs.
Only when necessary does it break at word level.

ADVANTAGES

  • Preserves semantic meaning
  • Respects natural boundaries
  • Good balance of size/context
  • Handles various text structures

DISADVANTAGES

  • More complex implementation
  • Variable chunk sizes
  • Requires parameter tuning
  • Language-dependent separators
D
Document-aware Chunking
Understands document structure (headers, sections, lists, tables) and creates chunks that respect the logical organization and hierarchy of the content.
